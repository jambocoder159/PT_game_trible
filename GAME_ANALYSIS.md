# PT Game Trible (三消挑戰) - 遊戲機制深度分析與產品迭代路線圖

---

## Context

PT Game Trible 是一款 HTML5 Canvas + JavaScript 建構的手機版三消益智遊戲，目前具備 8 種遊戲模式、30 關闖關戰役、RPG 生存系統、道具商城、排行榜等功能。本分析旨在從**玩法深度與樂趣**角度出發，對標業界頂尖三消及休閒遊戲，找出核心優缺點，並產出可執行的產品迭代路線圖。

---

## 一、核心玩法機制分析

### 1.1 核心循環 (Core Loop)

**現狀描述：**
- 玩家點擊方塊的不同區域執行三種操作：移到最上方（左 1/3）、直接移除（中 1/3）、插入最下方（右 1/3）
- 操作後自動偵測 3+ 同色方塊進行消除，觸發連鎖反應
- 每次成功消除增加 Combo 計數，失敗則歸零
- 5 種顏色：紅、藍、綠、黃、紫

**關鍵檔案：**
- `web/js/GameEngine.js` (3888 行) - 核心遊戲引擎
- `web/js/GameModes.js` - 模式配置
- `web/js/ItemSystem.js` - 道具系統
- `web/js/SkillSystem.js` - 技能系統
- `web/js/QuestMode.js` - 闖關模式
- `web/js/ShopSystem.js` - 商城系統

### 1.2 優點分析

| 優點 | 說明 | 對標競品 |
|------|------|----------|
| **獨特的三區域互動** | 一個方塊有 3 種操作方式，這在三消遊戲中非常罕見，提供了比 Candy Crush 的「交換」或 Royal Match 的「滑動」更高的策略深度 | 獨創 |
| **多模式設計** | 8 種模式覆蓋休閒（經典）、競速（限時）、策略（闖關）、成長（RPG），滿足不同玩家需求 | 類似 Archero 的多層次體驗 |
| **RPG 生存模式** | 結合等級成長、技能選擇、倒計時壓力、黑色方塊挑戰，是最有深度的模式 | 類似 Vampire Survivors 的 Roguelike 概念 |
| **闖關戰役設計** | 30 關 3 章節，每關有獨特敵人、血量、限制條件，提供明確的進度感 | 類似 Candy Crush 的關卡地圖 |
| **Combo/連鎖系統** | 連擊里程碑獎勵（5/10/15/20/30 連擊）和連鎖倍率，鼓勵高技巧操作 | 類似 Puzzle & Dragons 的 Combo 系統 |
| **道具系統完整** | 9 種道具從基礎到史詩級，稀有度分級，提供戰術選擇 | 類似 Royal Match 的 Booster 系統 |
| **雲端社交功能** | Supabase 後端支持排行榜、OAuth 登入、進度同步 | 業界標準 |

### 1.3 缺點分析（依嚴重程度排序）

#### 致命問題 (P0)

| 問題 | 影響 | 詳情 |
|------|------|------|
| **核心操作不夠直覺** | 新玩家流失率高 | 「點擊方塊的不同 1/3 區域」在手機上觸控精度不足，容易誤操作。對比 Candy Crush 的「交換」和 Royal Match 的「滑動」，本遊戲的操作門檻過高。教學雖然有 TutorialSystem，但 step-by-step 的靜態教學不如引導式互動教學有效 |
| **缺乏 Meta 層（裝飾/敘事）** | 長期留存差 | Royal Match 的城堡裝飾、Gardenscapes 的花園敘事，是三消遊戲留存的核心驅動力。目前遊戲完全沒有 Meta 層，玩家缺乏「為什麼要繼續玩」的情感動力 |
| **經濟系統未閉環** | 變現基礎薄弱 | 只有 RPG 模式能賺金幣（`baseScore/4`），其他 6 個模式完全沒有經濟產出。1000 金幣的測試預設值意味著經濟系統尚未正式設計 |

#### 嚴重問題 (P1)

| 問題 | 影響 | 詳情 |
|------|------|------|
| **單排模式過於單調** | 首次體驗差 | 經典/教學模式只有 1 列，橫向消除不可用，策略空間極其有限，容易讓新手誤以為整個遊戲就是如此 |
| **第 2、3 章已寫好但鎖住** | 內容浪費 | `QuestMode.js` 中 chapters 設定第 2、3 章 `levels: 0`，但 `GameModes.js` 已定義了 11-30 關的完整數據 |
| **闖關限制重複性高** | 挑戰感不足 | 30 關中，限制條件主要是 `noDamageColors`、`damageOnlyColors`、`minComboForDamage` 的排列組合，缺乏機制變化（`requireHorizontalMatch` 已寫但被註解掉） |
| **RPG 技能同質化** | 選擇無感 | 8 個技能中 5 個是顏色加成（紅/藍/綠/黃/紫），玩法影響相似，選擇無明顯差異感 |
| **缺乏日常回訪機制** | 日活低 | 無每日獎勵、無體力系統、無每日任務、無登入獎勵，玩家沒有理由「每天回來」 |

#### 中等問題 (P2)

| 問題 | 影響 | 詳情 |
|------|------|------|
| **無成就系統** | 缺乏中長期目標和收集驅動力 | - |
| **排行榜缺乏社交互動** | 只有全球排名，沒有好友排名、公會排名 | - |
| **無離線進度** | 不像 idle 遊戲，關掉就完全停止 | - |
| **手機觸覺回饋缺失** | 消除、Combo、道具使用都沒有振動回饋 | - |
| **黑色方塊只在生存模式** | 這個有趣的機制被限制在單一模式中，未充分利用 | - |

### 2.3 本遊戲的獨特定位

**差異化優勢：** 「三區域互動」是市面上沒有的獨創三消機制，天然適合策略型玩家。

**建議定位：** **策略型三消 RPG** — 介於 Candy Crush（純休閒）和 Puzzle & Dragons（重度策略）之間。史前部落主題在三消品類中幾乎空白，是一個藍海定位。

## 三、產品迭代路線圖

### Phase 1：核心體驗優化（短期 1-2 週）

> 目標：讓現有玩法更好玩、更易上手

#### 1.1 操作體驗改善
- **改善觸控區域辨識度**：在方塊上添加視覺提示（細微的分區線或圖標），讓玩家清楚知道點擊不同區域的效果
- **添加操作預覽**：長按方塊時顯示該操作的效果預覽（方塊會移到哪裡）
- **觸覺回饋**：消除時添加手機振動（`navigator.vibrate()`）
- **修改檔案**：`GameEngine.js` 的點擊處理和渲染邏輯

#### 1.2 開放已有內容
- **解鎖第 2、3 章**：`QuestMode.js` 中將 `levels: 0` 改為 `levels: 10`
- **啟用 requireHorizontalMatch**：取消 `GameModes.js` 中被註解的限制條件，增加闖關變化
- **修改檔案**：`QuestMode.js`, `GameModes.js`

#### 1.3 經濟系統閉環
- **所有模式都能賺金幣**：Classic/Double/Triple 模式結束時根據分數給予金幣獎勵
- **移除測試預設值**：`ShopSystem.js` 中 `defaultCurrency: 1000` 改為 `0`，新玩家通過遊玩獲得初始金幣
- **修改檔案**：`GameEngine.js`, `ShopSystem.js`

### Phase 2：留存機制建立（中期 2-4 週）

> 目標：讓玩家有理由每天回來

#### 2.1 每日系統
- **每日挑戰**：每天生成一個特殊條件的關卡（如「只用藍色方塊通關」），完成給予額外金幣
- **每日登入獎勵**：7 天循環獎勵（金幣、道具、稀有道具）

#### 2.2 成就系統
- **里程碑成就**：首次 10 連擊、首次通關第 10 關、累計消除 1000 個方塊等
- **頭銜系統**：達成特定成就解鎖頭銜，顯示在排行榜上

#### 2.3 闖關模式豐富化
- **新增限制機制**：實作 `requireHorizontalMatch`；新增「限時闖關」「限定道具」等限制
- **Boss 機制**：每章最後一關添加 Boss 特殊攻擊（如每 3 回合翻轉方塊顏色）
- **星級評價**：每關根據剩餘步數/分數給予 1-3 星，鼓勵重玩

#### 2.4 RPG 技能多樣化
- **新增主動技能**：如「時間凍結」（暫停倒計時 3 秒）、「全場洗牌」、「指定消除某色」
- **技能組合效果**：某些技能搭配有額外效果（如紅+藍加成同時存在時觸發「雙色爆發」）

### Phase 3：Meta 層與社交（中長期 4-8 週）

> 目標：建立情感連結和社交黏性

#### 3.1 部落建設 Meta
- **概念**：玩家經營一個部落村莊，通關/得分獲得「建設點數」，用來升級建築（篝火、圖騰、長老小屋等）
- **效果**：不同建築提供遊戲內 buff（如「篝火 Lv3：所有模式基礎分 +10%」）
- **情感驅動**：視覺化的部落成長提供長期目標和歸屬感
- **參考**：Royal Match 的城堡裝飾系統（$3B+ 營收的核心留存驅動力）

#### 3.2 社交功能增強
- **好友系統**：添加好友、好友排行榜、互贈體力/道具
- **公會/部落聯盟**：多人組隊，公會排行榜，公會任務

#### 3.3 賽季系統
- **賽季通行證**：每月重置，免費/付費雙軌獎勵（參考 Royal Match 的 $9.99 Battle Pass）
- **賽季主題**：每個賽季有不同的視覺主題和限定道具

### Phase 4：進階深度（長期 8+ 週）

> 目標：為核心玩家提供更深的遊戲體驗

#### 4.1 PvP 模式
- **即時對戰**：兩名玩家同時在相同方塊序列上操作，看誰分數更高
- **Snap 機制**（借鑒 Marvel Snap）：對戰中可以「加倍」賭注

#### 4.2 無盡 Roguelike 模式
- 每層隨機生成條件，每 5 層選擇一個永久 buff，看能走多遠
- **參考**：Archero 的 Roguelike 循環 + Vampire Survivors 的成長感

#### 4.3 特殊方塊系統
- **新方塊類型**：彩虹方塊（萬能色）、炸彈方塊（範圍消除）、冰凍方塊（需消除兩次）
- **方塊融合**：消除 5+ 時生成特殊方塊，增加策略層次
- **參考**：Candy Crush 的特殊糖果系統

## 五、核心結論

**本遊戲最大的資產是「三區域互動」的獨創機制**，這在三消遊戲市場中是真正的差異化。但目前的包裝（UI/UX、Meta 層、留存機制）遠不及這個核心機制的潛力。

**最關鍵的三件事：**
1. **讓操作更直覺**（Phase 1）— 降低獨創機制的學習門檻
2. **開放已有內容**（Phase 1）— 零成本增加 20 關遊戲內容
3. **建立 Meta 層**（Phase 3）— 這是 Royal Match 和 Gardenscapes 證明的、三消遊戲從「好玩」到「離不開」的關鍵

**市場機會：** 史前部落主題在三消品類中幾乎空白（Primitive Era: 10000 BC 是 SLG 而非三消），結合獨創的三區域互動機制，本遊戲有潛力在「策略型三消 RPG」這個利基市場建立獨特定位。
