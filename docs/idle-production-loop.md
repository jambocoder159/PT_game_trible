# 放置模式生產循環 — 設計規格

## 核心循環

```
方塊消除 → 魔法瓶累積能量 → 貓咪接單製作甜點 → 製作完成 → 售出獲得金幣/素材/經驗
```

---

## 1. 魔法瓶（能量系統）

### 基本規則
- 消除方塊時，對應顏色的能量球飛入魔法瓶
- 瓶子能量達到甜點成本即可開始製作（**不需要滿瓶**）
- 滿瓶 = 容量上限，不是製作門檻
- 瓶子滿了代表玩家應該回來收、排程、升級

### 範例
```
烘焙瓶目前 95 / 100 能量
奶油小餐包成本 30 能量
=> 可製作 3 個，剩 5 能量
```

### 瓶子等級效果
- 等級提升 → 容量增加
- 高等級解鎖更高階甜點
- 容量公式沿用現有：Lv1=100, Lv10=1850

---

## 2. 甜點（產品）

### 甜點定義
每個甜點需要：
- `energyCost` — 消耗的魔法瓶能量
- `craftDuration` — 製作所需時間（秒）
- `sellPrice` — 售出金幣

- 低階甜點：快速回收，適合頻繁操作

- 玩家決策：做便宜的快收錢 vs 做貴的等更久

## 3. 貓咪（工人）

### 基本規則
- 每隻貓咪同時只能製作 1 個甜點
- 隊伍最多 3 隻 = 同時 3 條產線
- 貓咪等級影響製作速度

### 速度公式
```
實際製作時間 = 基礎製作時間 / 貓咪效率倍率

效率倍率 = 1.0 + (catLevel - 1) * 0.02
```

| 貓咪等級 | 效率倍率 | 20s 甜點實際時間 |
|---------|---------|----------------|
| Lv.1 | 100% | 20s |
| Lv.10 | 118% | 17s |
| Lv.20 | 138% | 14.5s |
| Lv.30 | 158% | 12.7s |
| Lv.50 | 198% | 10.1s |

### 未來擴充（不在第一版）
- 稀有貓咪有特定甜點加成
- 同屬性貓咪搭配同色魔法瓶有加成
- 貓咪技能（減少材料、增加產出等）

---

## 4. 展示櫃（完成品）

### 基本規則
- 甜點製作完成後進入展示櫃
- 展示櫃有容量上限（初始 10，可升級）
- 玩家手動點擊售出（之後加自動售出）
- 售出時金幣爆出動畫

### 爆擊系統
- 售出時 15% 機率觸發「大成功」
- 大成功：+50% 金幣，特殊動畫效果

### 自動售出（後續版本）
- 通關 1-5 解鎖自動售出
- 自動售出不觸發爆擊
- 手動售出保留爆擊機會 → 鼓勵玩家回來操作

---

## 5. 離線補算

### 規則
- 關閉 app 時，正在製作的甜點繼續計時
- 回來時補算所有已完成的甜點，放入展示櫃
- 魔法瓶能量不會離線增長（需要消除方塊）
- 體力照常離線恢復

### 補算公式
```
離線時間 = now - lastActiveTime
對每個進行中的製作：
  if 離線時間 >= 剩餘製作時間:
    標記為完成，放入展示櫃
```

## 6. UI 佈局

### 主畫面三段式
```
┌─────────────────────────┐
│  PlayerInfoBar           │
├─────────────────────────┤
│  貓咪工作區              │  ← 演出：貓咪揉麵/烘烤/裝飾
│  製作進度條 x3           │  ← 每隻貓一條進度
├─────────────────────────┤
│  ☀️💧🍃⭐🌙 魔法瓶能量條 │  ← 達標時發光 + 「可製作」
├─────────────────────────┤
│  4×7 消除棋盤            │
├─────────────────────────┤
│  展示櫃：🍞🍞🧁🍰       │  ← 完成品堆積 + 售出按鈕
├─────────────────────────┤
│  底部導航                │
└─────────────────────────┘
```

### 演出流程
1. 消除方塊 → 能量球飛入魔法瓶
2. 魔法瓶達標 → 瓶口發光，出現「可製作」提示
3. 玩家點擊（或自動派工）→ 貓咪開始揉麵/烘烤/裝飾
4. 製作完成 → 甜點圖示堆在展示櫃
5. 點擊售出 → 甜點飛向收銀台，金幣爆出
6. 若爆擊 → 顯示「大成功」+ 額外金幣

## 7. 數據模型變更

### 新增 `ProductionSlot`
```dart
class ProductionSlot {
  String catId;           // 正在工作的貓咪
  String dessertId;       // 正在製作的甜點
  BlockColor sourceColor; // 消耗哪個瓶子的能量
  DateTime startTime;     // 開始時間
  int durationMs;         // 製作時間（毫秒）
  bool isComplete;        // 是否已完成
}
```

### 新增 `DisplayCase`（展示櫃）
```dart
class DisplayCase {
  Map<String, int> desserts;  // dessertId → 數量
  int maxCapacity;            // 展示櫃上限
  int get totalCount;         // 目前總數
  bool get isFull;
}
```

### 修改 `DessertRecipe`
```dart
// 新增欄位
int craftDurationSec;    // 製作時間（秒）
BlockColor? sourceColor; // 對應的魔法瓶顏色（null = 任意）
```

### 新增 `ProductionProvider`
```dart
class ProductionProvider extends ChangeNotifier {
  List<ProductionSlot> activeSlots;   // 進行中的製作（最多 3）
  DisplayCase displayCase;            // 展示櫃

  void startProduction(catId, dessertId, sourceColor);
  void tick();                        // 檢查完成狀態
  void sellAll();                     // 售出展示櫃全部
  void sellOne(dessertId);            // 售出單一甜點
  void recoverOffline(lastActiveTime); // 離線補算
}
```

### 保留不動
- `BottleProvider` — 能量累積邏輯不變
- `IdleProvider` — 消除邏輯不變
- `PlayerProvider` — 金幣/經驗操作不變

### 移除 / 棄用
- `CraftingProvider` 的食材合成系統（已有遷移機制）
- 食材相關 UI（crafting_panel.dart 的食材區塊）
