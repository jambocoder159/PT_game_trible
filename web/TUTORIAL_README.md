# 三消挑戰 - 教學系統

## 功能概述

這個教學系統為新手玩家提供了完整的互動式學習體驗，幫助他們快速掌握遊戲的所有操作和機制。

## 主要特色

### 🎓 互動式教學
- **逐步引導**：18個精心設計的教學步驟
- **實際操作**：玩家需要親自執行每個操作來進行下一步
- **即時反饋**：操作完成後自動進入下一個教學步驟

### 🎯 視覺化指導
- **高亮顯示**：重要元素會被高亮標示
- **對話框定位**：教學對話框會智能定位到相關元素附近
- **進度指示器**：底部顯示當前教學進度

### 📱 響應式設計
- **自適應布局**：支援各種螢幕尺寸
- **觸控友好**：適合手機和平板操作
- **鍵盤支援**：ESC鍵可跳過教學

## 教學內容

### 基礎操作
1. **遊戲介面介紹** - 認識遊戲畫面各個區域
2. **分數系統** - 了解分數顯示和計算方式
3. **連擊機制** - 學習連擊系統的重要性
4. **行動點數** - 理解行動點數的作用
5. **方塊移動** - 實際練習移動方塊
6. **方塊消除** - 學習如何創造三消組合

### 進階技能
7. **技能系統介紹** - 了解三種技能的作用
8. **移除技能** - 實際使用移除單個方塊技能
9. **重骰技能** - 學習重骰下個方塊的技能
10. **變色技能** - 了解最強大的變色板面技能

### 策略指導
11. **連鎖反應** - 理解連鎖消除的機制
12. **連擊分數** - 掌握連擊分數系統
13. **策略建議** - 獲得遊戲策略提示

## 技術實現

### 核心類別：TutorialSystem
```javascript
class TutorialSystem {
    constructor(gameEngine)     // 初始化教學系統
    start()                     // 開始教學
    stop()                      // 停止教學
    skip()                      // 跳過教學
    nextStep()                  // 下一步
    previousStep()              // 上一步
}
```

### 教學步驟結構
```javascript
{
    title: "步驟標題",
    content: "詳細說明",
    target: "#元素選擇器",      // 要高亮的元素
    position: "對話框位置",      // top/bottom/left/right/center
    action: "等待動作",          // highlight/waitForMove/waitForMatch等
    autoNext: true/false,       // 是否自動進入下一步
    instruction: "操作提示"      // 額外的操作說明
}
```

### 互動監聽
- **移動操作監聽**：等待玩家完成方塊移動
- **消除監聽**：等待玩家創造三消組合
- **技能使用監聽**：等待玩家使用特定技能
- **按鈕點擊監聽**：監聽技能按鈕和重骰操作

## 使用方式

### 從主選單進入
1. 在主選單點擊「遊戲教學」按鈕
2. 系統會自動載入教學模式
3. 跟隨對話框指示完成每個步驟

### 教學控制
- **下一步**：點擊「下一步」按鈕或完成要求的操作
- **上一步**：點擊「上一步」按鈕回到前一個步驟
- **跳過教學**：點擊右上角「跳過教學」按鈕
- **ESC鍵**：快速跳過教學

### 完成教學
- 完成所有步驟後會顯示恭喜訊息
- 可選擇返回主選單開始正式遊戲


## 自定義和擴展

### 添加新的教學步驟
在 `TutorialSystem.js` 的 `setupTutorialSteps()` 方法中添加新的步驟對象。

### 修改教學內容
編輯現有步驟的 `title` 和 `content` 屬性來更新教學文字。

### 添加新的互動類型
在 `handleStepAction()` 方法中添加新的 action 類型和對應的處理邏輯。

## 注意事項

1. **遊戲引擎依賴**：教學系統需要完整的遊戲引擎支援
2. **DOM元素**：確保所有被引用的DOM元素都存在
3. **事件處理**：教學期間會暫時覆蓋部分遊戲事件處理器
4. **記憶體管理**：教學結束後會自動清理事件監聽器

行動點數
每次操作會消耗行動點數。當行動點數歸零時遊戲結束，所以要謹慎使用！
會是執行步驟後沒有消除任何方塊才會扣行動點
